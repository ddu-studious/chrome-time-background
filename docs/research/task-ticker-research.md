# 未完成任务滚动提醒条调研

> **调研日期**: 2026-02-08  
> **调研目标**: 在现有 Tech Ticker 附近增加「未完成任务提醒滚动条」，让用户在新标签页不打开侧边栏也能看到待办  
> **灵感来源**: 现有 Tech Ticker 混合双层模式的迷你滚动栏效果

---

## 一、需求分析

### 1.1 痛点

- 当侧边栏折叠时，用户无法感知有多少未完成任务
- 需要一个轻量级的「提醒带」，不干扰主要内容但保持存在感
- 与现有 Tech Ticker 协调，不显得杂乱

### 1.2 数据源

- `chrome.storage.local` 中的 `memos` 数组
- 筛选条件：`completed === false`
- 可选：按优先级/截止日期排序
- 可选：仅显示今日到期 / 已过期的任务

### 1.3 位置方案

| 位置 | 优点 | 缺点 |
|------|------|------|
| Tech Ticker 下方 | 自然延伸，视觉统一 | 可能占太多纵向空间 |
| Tech Ticker 上方 | 突出提醒优先级 | 打断整体视觉流 |
| 时钟下方 | 位于视觉中心 | 分散时钟注意力 |
| 页面底部 | 不干扰主内容 | 容易被忽略 |
| **Tech Ticker 同层** | 最紧凑，作为 ticker 的第三层 | 实现复杂度稍高 |

**推荐**：放置在 Tech Ticker 的下方，作为独立的一行，与 ticker 视觉上一体但功能独立。

---

## 二、5 种方案调研

### 方案 1：水平连续滚动（Marquee 模式）

**描述**：所有未完成任务标题在水平方向连续循环滚动，类似新闻跑马灯。

**技术要点**：
```css
@keyframes taskMarquee {
    from { transform: translateX(0); }
    to { transform: translateX(-50%); }
}
.task-ticker-track {
    display: flex;
    animation: taskMarquee 30s linear infinite;
    will-change: transform;
}
```

**优点**：
- 与现有 Tech Ticker 下层完全一致，视觉统一
- 纯 CSS 动画，性能极佳
- 渐变遮罩实现平滑边缘

**缺点**：
- 信息阅读不方便（需等待滚动到想看的位置）
- 长任务标题可能太长

**推荐度**：⭐⭐⭐⭐

---

### 方案 2：优先级色带 + 滚动

**描述**：每个任务标题前带有优先级色点，高优先级任务排在前面，整体水平连续滚动。过期任务带红色脉冲效果。

**技术要点**：
```css
.task-item-dot {
    width: 6px; height: 6px;
    border-radius: 50%;
    background: var(--priority-color);
}
.task-item-dot.overdue {
    animation: overdueGlow 1.5s infinite;
}
@keyframes overdueGlow {
    0%, 100% { box-shadow: 0 0 0 0 rgba(255,60,60,0.6); }
    50% { box-shadow: 0 0 0 4px rgba(255,60,60,0); }
}
```

**优点**：
- 一目了然的优先级信息
- 过期任务的脉冲效果非常醒目
- 视觉层次丰富

**缺点**：
- 色点过多时可能视觉拥挤
- 需要计算优先级排序

**推荐度**：⭐⭐⭐⭐⭐

---

### 方案 3：任务计数摘要 + 悬浮展开

**描述**：平时只显示一行紧凑的统计摘要（如「3 个任务今日到期 · 2 个已过期」），鼠标悬停时向下展开显示完整任务列表。

**技术要点**：
```css
.task-summary-bar { height: 28px; transition: height 0.3s; }
.task-summary-bar:hover { height: auto; max-height: 200px; }
```

**优点**：
- 默认最小化，不占空间
- 悬停展开交互直观
- 信息密度可控

**缺点**：
- 没有动态感，静态展示
- 需要鼠标悬停才能看到详情

**推荐度**：⭐⭐⭐⭐

---

### 方案 4：分段进度 + 滚动混合

**描述**：左侧固定展示一个环形进度（完成率），右侧是任务标题的水平滚动。形成「数据 + 流」的布局。

**技术要点**：
```html
<div class="task-bar">
    <div class="task-bar-stats">✅ 5/12</div>
    <div class="task-bar-scroll">
        <!-- marquee 滚动 -->
    </div>
</div>
```

**优点**：
- 左侧统计一目了然
- 右侧滚动展示具体任务
- 信息完整度高

**缺点**：
- 布局稍复杂
- 左右两块风格需统一

**推荐度**：⭐⭐⭐⭐⭐

---

### 方案 5：垂直轮播卡片

**描述**：单条垂直轮播，每次展示一个任务（标题 + 截止日期 + 优先级），自动向上滚动切换。

**技术要点**：
```css
@keyframes slideUp {
    0% { transform: translateY(0); opacity: 1; }
    90% { transform: translateY(0); opacity: 1; }
    100% { transform: translateY(-100%); opacity: 0; }
}
```

**优点**：
- 每个任务展示完整
- 动画优雅
- 可带操作按钮（快速完成）

**缺点**：
- 信息密度低（一次只看一条）
- 类似旧的单条 ticker 模式

**推荐度**：⭐⭐⭐

## 四、推荐方案

### 首选：方案 4（分段进度 + 滚动混合）

理由：
1. 左侧统计摘要 + 右侧任务流，信息完整
2. 与 Tech Ticker 的双层模式风格高度一致
3. 一行即可展示关键信息
4. 点击可快速跳转到任务详情

### 备选：方案 2（优先级色带 + 滚动）

理由：
1. 视觉效果突出，优先级一目了然
2. 过期任务脉冲效果非常醒目
3. 实现相对简单
